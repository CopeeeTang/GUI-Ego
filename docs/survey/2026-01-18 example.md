## 3. Google Dynamic View Prompt 设计

### 3.1 智能眼镜 UI 约束

| 约束 | 说明 |
|------|------|
| 显示方式 | 绿色亮文字 (Green bright text) |
| 视野限制 | 小视野区域，需最小化信息量 |

## 4. 长期 vs 短期交互模式分析

### 4.1 定义

| 类型 | 时长 | 特征 | 示例 |
|------|------|------|------|
| **短期交互** | 秒级 - 分钟级 | 单次查询，快速决策 | 价格查询、物体识别、方向指引 |
| **长期交互** | 分钟级 - 小时级 | 持续陪伴，状态追踪 | 烹饪指导、学习辅助、运动追踪 |

### 4.2 UI 设计差异

| 维度 | 短期交互 | 长期交互 |
|------|----------|----------|
| **信息密度** | 高 (快速获取信息) | 低 (最小干扰) |
| **持久性** | 短暂显示，自动消失 | 持续存在，后台状态 |
| **交互频率** | 高 (即时响应) | 低 (定期更新) |
| **UI 复杂度** | 可较复杂 (专注查看) | 必须极简 (peripheral) |
| **唤醒方式** | 主动触发 (语音/注视) | 被动提醒 (条件触发) |

### 4.3 从数据集提取的模式

**短期交互模式** (从购物场景):
```
用户注视 → 物体检测 → 信息叠加 → 决策辅助 → 自动消失
[0.5s]      [0.3s]      [显示中]     [用户确认]   [5s后]
```

**长期交互模式** (从学习场景):
```
场景进入 → 模式建议 → 用户确认 → 后台运行 → 周期提醒
[检测]     [弹出]      [一键]      [最小化]     [间隔触发]
```

## 5. 免打扰时段分析 (Do-Not-Disturb)

### 5.1 从数据集识别的禁止推荐时段

| 场景类型 | 典型情况 | 处理策略 |
|----------|----------|----------|
| 社交场景 | 与人交流、与店员沟通 | 完全禁用 UI |
| 安全场景 | 过马路、下楼梯 | 仅保留导航指引 |
| 专注场景 | 自习中、睡觉中 | 仅紧急通知 |
| 环境因素 | 人多拥挤、环境安静 | 静音+简化 UI |

### 5.2 唤醒时机建议

| 触发条件 | 推荐类型 | UI 响应 |
|----------|----------|---------|
| 场景切换 (进入超市) | context_aware | 主动弹出选项 |
| 物体注视 (>2秒) | object_identification | AR 信息叠加 |
| 语音关键词 | 所有类型 | 全功能响应 |
| 生理信号 (疲劳) | task_assistance | 休息提醒 |
| 时间条件 (预约) | contextual_memory | 提前提醒 |

## 6. Gap 分析：WebUI vs 智能眼镜 UI

### 6.1 Google Generative UI 论文回顾

| 维度 | 论文方法 | 智能眼镜需求 | Gap |
|------|----------|--------------|-----|
| **输出形式** | 富 HTML 页面 | 最小化 AR 叠加 | 需极简化适配 |
| **交互方式** | 鼠标/触摸 | 语音/注视/手势 | 输入模态完全不同 |
| **显示空间** | 全屏幕 | 小视野区域 | 信息密度限制 |
| **上下文** | 静态网页 | 动态真实世界 | 需实时环境理解 |
| **注意力** | 专注屏幕 | 分散到环境 | 需注意力感知 |

### 6.2 核心 Gap

1. **无实时多模态融合**: Google GenUI 基于文本 prompt，不支持视觉/语音/生理信号融合
2. **无空间锚定**: Web UI 不需要 AR 定位，智能眼镜需要将 UI 锚定到物理对象
3. **无注意力感知**: Web UI 假设用户专注，智能眼镜需要判断用户注意力分配
4. **无免打扰机制**: Web UI 总是显示，智能眼镜需要知道何时隐藏
5. **无渐进式披露**: Web UI 一次展示全部，智能眼镜需要分层展示

### 6.3 需要的扩展

```
Google GenUI                    Smart Glasses GenUI
─────────────                   ───────────────────
Text Prompt  ─────────────────▶ Multi-modal Context
                                (视觉 + 语音 + 注视 + 生理)

HTML Output  ─────────────────▶ AR Component Output
                                (空间锚定 + 极简样式)

Static Page  ─────────────────▶ Dynamic Overlay
                                (注意力感知 + 免打扰)

User Click   ─────────────────▶ Gaze + Voice + Gesture
                                (多模态输入融合)
```

## 7. A2UI 调研

### 7.1 A2UI 概述

**GitHub**: https://github.com/google/A2UI

**定义**: A2UI (Agent-to-User Interface) 是一个开源协议和库集合，允许 AI Agent 生成丰富、交互式的 UI，并在各种平台上原生渲染。

### 7.2 核心设计理念

| 原则 | 说明 |
|------|------|
| **Security First** | UI 消息是声明式 JSON 数据，不是可执行代码，防止 Agent 注入 JavaScript/HTML |
| **LLM-Friendly** | UI 表示为带 ID 引用的扁平组件列表，便于 LLM 增量生成 |
| **Framework-Agnostic** | 同一 JSON payload 可在 Web Components、Flutter、React、SwiftUI 等不同框架渲染 |
| **Flexibility** | 支持开放注册表模式，开发者可将服务端类型映射到自定义客户端实现 |

### 7.3 工作流程

```
┌──────────────┐    ┌──────────────┐    ┌──────────────┐    ┌──────────────┐
│  Generation  │ →  │  Transport   │ →  │  Resolution  │ →  │  Rendering   │
│  (LLM生成)   │    │  (JSON传输)  │    │  (解析JSON)  │    │  (原生渲染)  │
└──────────────┘    └──────────────┘    └──────────────┘    └──────────────┘
     Agent              A2A/SSE           A2UI Renderer      Native Components
```

**通信方式**: JSON Lines (JSONL) 流，通常通过 Server-Sent Events (SSE)

**消息类型**:
- `surfaceUpdate`: 组件定义
- `dataModelUpdate`: 数据更新
- `beginRendering`: 开始渲染信号
- `deleteSurface`: 移除 UI 区域


### 7.5 与智能眼镜的适用性分析

| 维度 | A2UI 现状 | 智能眼镜需求 | 适配可行性 |
|------|-----------|--------------|------------|
| **平台支持** | Web, Mobile, Desktop | AR/Smart Glasses | ⚠️ 需开发新 Renderer |
| **输入方式** | 鼠标/触摸事件 | 语音/注视/手势 | ⚠️ 需扩展 userAction |
| **空间锚定** | 不支持 | 必需 | ❌ 需从头设计 |
| **显示约束** | 任意尺寸 | 小视野/绿色文字 | ⚠️ 需定制组件 |
| **注意力感知** | 不支持 | 必需 | ❌ 需扩展协议 |

### 7.6 关键结论

**可借鉴点**:
1. **声明式 JSON 协议**: 安全且 LLM 友好，可作为智能眼镜 UI 的基础协议
2. **框架无关性**: 可以为智能眼镜开发专用 Renderer
3. **增量更新机制**: 适合流式场景，可用于实时 AR 更新

**需要扩展的方向**:
1. **空间坐标系统**: 添加 3D 位置/锚点字段
2. **注视触发组件**: 添加 `gaze-aware` 属性
3. **智能眼镜专用组件**: AR Label、Direction Arrow、Minimal Card 等
4. **免打扰机制**: 添加 `priority` 和 `context` 字段

### 7.7 VLA-like 模拟器调研

**现有工具**:
| 工具 | 类型 | 适用性 |
|------|------|--------|
| Unity AR Foundation | 开发框架 | 需要大量开发 |
| Varjo XR Simulator | 商业模拟器 | 高保真但昂贵 |
| WebXR | Web 标准 | 可快速原型但功能有限 |
| A2UI + Custom Renderer | 可行方案 | 基于 A2UI 协议开发 AR Renderer |

**Gap**: 目前没有专门针对智能眼镜 UI 的 VLA-like 模拟器，但 A2UI 提供了一个可扩展的基础
